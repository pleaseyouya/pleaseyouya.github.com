记Proguard过程中踩过的几个坑

今天在打release包，加混淆。打包过程中，报了几个错。

Error:ProGuard: Can’t process class [com/example/msearchpublic/BuildConfig.class (Unsupported class version number [52.0] (maximum 51.0, Java 1.7)))

看了下ProGuard的版本，切换到D:\ADT\sdk\tools\proguard\lib，


Proguard的版本是4.7，而根据Proguard版本的JDK版本的关系

J2SE 8 = 52,
J2SE 7 = 51,
J2SE 6.0 = 50,
J2SE 5.0 = 49,
JDK 1.4 = 48,
JDK 1.3 = 47,
JDK 1.2 = 46,
JDK 1.1 = 45

我的JDK是1.8版本的，所以，需要Proguard更新到5.2。
更新的方法是，下载最新版的Proguard包，解压之后，将proguard文件夹直接覆盖D:\ADT\sdk\tools\proguard即可。
